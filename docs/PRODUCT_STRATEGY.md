# MemoSpark 产品成熟化策略

> 调研日期：2026-06-28  
> 方法：源码审计（前后端 + 数据模型 + SRS/就绪度引擎）＋ 竞品调研 ＋ PM 框架（pm-skills: product-strategy / value-proposition / north-star-metric / opportunity-solution-tree / prioritize-features）。  
> 用户假设：**你本人**，正在备战 SWE/后端方向面试（截图技能：Java、算法、英语）。  
> 关键立场：这是**单人深度工具**，不是面向大众的 SaaS。北极星不是 DAU/MAU，而是「**帮你拿到 offer**」的领先指标。

---

## 0. TL;DR

1. 你的真问题**不是功能少**，技术底子其实在「成熟产品」水准（SM-2 SRS、JWT、Flyway、CI/CD、PWA + 小程序 + MCP、还有差异化的 JD→技能→就绪度）。
2. 它「像 demo」的根因是 **闭环断了 + 核心指标不可信**：
   - **闭环断了**：`TargetSkillService` 里，AI 已经把 JD 拆成了带 `topics` + `suggestedCardCount` 的「学习牌组」，但代码只取了个权重，**把牌组和卡片全丢了**。于是「JD → 定制学习计划 → 卡片 → 复习」这条差异化主线只剩一个孤立的自评列表。
   - **不可信**：面试就绪度里 `skillCoverage` 占 50% 权重，而它完全来自你**手点 0-5 自评**。自己给自己打的分，不能给你"我真的准备好了"的信心。
3. 成熟化三步走（按杠杆排序）：
   - **P0 接闭环**：JD→AI 牌组**真正落库成卡**，薄弱技能"去强化"直达对应牌组。（复用已有 AI 输出，改动最小，价值最大）
   - **P1 让就绪度有证据**：用复习正确率/题目通过率自动驱动掌握度；SM-2 → **FSRS**。
   - **P2 加输出训练**：AI **模拟面试**（按 JD 出题、开口/打字作答、结构化点评、评分回流就绪度）——这是真正"帮你找到工作"的部分。

## 1. 这类产品的底层逻辑（调研结论）

把 Anki/FSRS、面试鸭、牛客、Final Round AI、Exponent、Huntr 拆开看，**成熟产品共享 7 条底层逻辑**：

1. **造卡/找内容的摩擦是头号杀手**。证据：业界共识「大多数人用 Anki 两周内放弃，而放弃几乎都发生在*造卡*阶段，不是复习。复习很轻松，制造卡片才是那堵墙」。→ 成熟产品要么给 **curated 题库**（面试鸭：近百题库、几千高频题），要么用 **AI 自动生成**（把读到的内容一键成卡）。
2. **断签→积压→愧疚→弃用** 是第二大流失。错过几天 → 到期卡雪崩 → 打开就有负罪感 → 干脆不打开。→ 需要**每日新卡上限 + 积压"重新规划"而非无脑堆积 + 去掉 streak 焦虑**。
3. **调度算法已换代**：SM-2（1980s 固定倍率，有"ease hell"）→ **FSRS**（建模 retrievability/stability/difficulty，设定目标留存率如 90%，**同样留存更少复习量**）。FSRS 需要复习历史来训练（约 400–1000 条后收益显著）。
4. **按目标个性化**：面试鸭/牛客都是**按岗位/方向分库**、**按高频排优先级**；JD 驱动的学习计划是更强的个性化。
5. **训练输出，不只背输入**。AI 模拟面试品类（Final Round、Exponent）统一逻辑：**传 JD/选岗位 → AI 出岗位定制题 → 开口答 → 即时点评(内容/结构/语速/举例) → 评分 + 回放/转写 → 重复**。覆盖行为题(STAR)、技术题、系统设计。价值在于**高频次 + 快反馈**。
6. **去情境化记忆 ≠ 真实能力**。能在卡片语境里背出，不代表面试现场能讲清。→ 把"复习"连到"应用/讲解"。
7. **连到真实求职漏斗**。Huntr 这类把"投递→面试→笔记→指标"全程跟踪，JD 自动高亮技能词。→ 就绪度只有连到真实结果才有意义。

> 一句话：成熟产品都在跑同一个闭环——**评估 → 计划 → 练习(输入复习 + 输出模拟) → 用证据度量 → 重新规划**。MemoSpark 现在有"评估(自评)"和"练习(通用复习)"，但**计划和度量是断的、不可信的**。

## 2. 现状诊断：为什么"像个人 demo"（带源码证据）

| # | 根因 | 证据 | 后果 |
|---|------|------|------|
| R1 | **差异化闭环断在最后一步** | `service/TargetSkillService.java:46-67`：AI 返回 `decks`（含 `topics`、`suggestedCardCount`），代码仅 `mapWeight(suggestedCardCount)`，丢弃 topics 与卡片生成 | "JD→技能"只产出一个**死的自评列表**；Dashboard"提升薄弱技能"点了只跳回目标页，没有可练的东西 |
| R2 | **就绪度不可信（自评驱动）** | `service/ReadinessService.java:39-50,66`：`skillCoverage` = 加权自评 0-5，且占 `overall` 的 **50%** | 分数是你自己点出来的，给不了"真的准备好了"的确定感 |
| R3 | **内容是通用内置库，未按目标生成** | `DecksPage` 全是 `BUILTIN`（Java/算法/英语/LeetCode Hot100）；JD 分析不产卡 | 看起来对谁都一样，没有"为我的目标岗位定制"的感觉 |
| R4 | **只练输入，没有输出闭环** | 全站只有"翻卡复习 + 代码评测"，无模拟面试/开口讲解/答案点评 | 最能决定 offer 的"会不会讲"完全没练 |
| R5 | **冷启动空洞 + 调度过时** | 截图：0 连续、0 复习、0% 留存、无目标；`SrsEngine.java` 纯 SM-2 | 首屏像空壳；断签后积压会劝退 |

**补充观察**

- 统计页 `StatsPage.tsx` 仅 ~3.7KB，大概率是基础计数；缺**遗忘曲线 / 就绪度趋势 / 预测校准**。
- AI 用得浅：只做 JD 拆解 + 评分兜底；**没用于生成卡片/出模拟题/讲解错因/每日教练**。
- 数据底子其实够：已有 `ReviewLog`（FSRS 训练数据）、`SrsSettings`、`CardProgress`、`ProblemNote`——升级不需要重建地基。

## 3. 定位 / JTBD / 价值主张 / 北极星

**Product Vision**  
> 让我对着某个具体岗位，能清楚知道"还差什么"、把它补上、并**用证据证明我真的会了**——走进面试间时是有底气的，而不是赌运气。

**JTBD（核心 Job Story）**  
> 当我**锁定一个目标岗位/公司、面试在 N 天后**时，  
> 我想要**知道该练什么、把它练熟、并验证我真的掌握了**，  
> 这样我能**自信地走进面试，不被意料之外的问题打懵**。

**价值主张（对比）**  
- 通用 SRS（Anki）：强调用法自由，但**造卡靠自己**、与"找工作"无关。  
- 题库（面试鸭/牛客）：内容多，但是**别人的题、按通用方向**，不针对"我的 JD"，且偏看题不偏"记牢 + 会讲"。  
- AI 模拟面试（Final Round/Exponent）：练输出强，但**不沉淀长期记忆**、不管你"知识点是否记得住"。  
- **MemoSpark 的位置 = 唯一把这三者缝起来的个人闭环**：JD 定制内容 → 间隔重复记牢 → 模拟面试会讲 → 证据化就绪度。这是"即使只有我一个人用也极好用"的支点。

**North Star Metric（单人版）**  
不要用 DAU。建议：

> **每周"证据化就绪度"净增**（Weekly Evidence-Based Readiness Gain）

- 它是 productivity game（帮你高效达成目标），不是 attention game。  
- 输入指标（Input Metrics，直接可推动）：  
  1. **当周经证据更新的技能数**（复习/做题/模拟带来的掌握度变化，而非自评）  
  2. **到期卡按时清空率**（记忆纪律，且不积压）  
  3. **当周模拟面试次数 × 平均得分提升**  
  4. **目标技能覆盖率**（有 JD、有卡、在练的技能 / JD 要求技能）

## 4. Opportunity Solution Tree

```
Outcome: 提高"对某目标岗位的、可信的面试就绪度"
├─ O1 造卡/找内容负担重
│   └─ S1 JD→AI 自动生成定制牌组+卡片（接回被丢弃的 decks）
│   └─ S2 面经/笔记/错题一键成卡（AI 抽取 atomic Q/A）
├─ O2 就绪度不可信（纯自评）
│   └─ S3 evidence-based mastery：复习正确率/题目通过率自动更新掌握度
│   └─ S4 SM-2 → FSRS，给出"目标留存率 + 预测遗忘"
├─ O3 只练输入不练输出
│   └─ S5 AI 模拟面试：JD 出题→作答→结构化点评→评分回流就绪度
│   └─ S6 "讲给我听"模式：开口/打字解释知识点，AI 评清晰度
├─ O4 断签积压愧疚
│   └─ S7 每日新卡上限 + 倒计时驱动的"今日最小必做" + 积压重排
├─ O5 与真实求职脱节
│   └─ S8 轻量投递/面试漏斗跟踪，把就绪度连到真实结果
```

## 5. 优先级 Roadmap（RICE 直觉排序，单人杠杆）

| 优先级 | 初始项 | 为什么先做 | 改动量 |
|--------|--------|-----------|--------|
| **P0** | **S1 接回 JD→牌组→卡片闭环** + 技能↔牌组关联 | 复用已写好的 AI 输出，**最小改动激活最大差异化**；让"提升薄弱技能"真正可点可练 | 中（后端 1 个 service + 1 个字段 + 前端跳转） |
| **P1** | **S3 证据化掌握度** + **S4 FSRS** | 让就绪度可信（这是"成熟感"的核心）；FSRS 数据基础已有(`ReviewLog`) | 中-大 |

## 6. 怎么集成（P0 落地，具体到文件/字段）

**目标**：粘贴 JD → 一键得到"为这个岗位定制的牌组 + 卡片" → 直接开练；薄弱技能一键跳到对应牌组。

1. **后端 · 接回被丢弃的牌组**（`TargetSkillService.analyzeAndPersist`）  
   - 现状：`decks[*]` 的 `topics`/`suggestedCardCount` 被丢，只存 `TargetSkill`。  
   - 改造：对每个 skill area，创建一个 `Deck`（`DeckType` 标记为 `TARGET`/`GENERATED`），调用 AI 用 `topics` 生成 `suggestedCardCount` 张 **atomic Q/A** 卡片落库；`TargetSkill` 增加 `deckId` 外键。
2. **数据模型**：`TargetSkill` 加 `deck_id`（Flyway 新增 V6 迁移）；`Deck` 增加来源标识与 `targetId`。
3. **前端 · 打通"去强化"**  
   - `DashboardPage.tsx:147-149` 的"提升薄弱技能"→ 跳 `"/review/" + skill.deckId`（现在跳的是目标页死路）。  
   - `TargetDetailPage` 的 `SkillRow` 增加"复习此技能(n张)"按钮 → `/review/:deckId`。
4. **就绪度顺势可信化（P1 衔接）**：`ReadinessService` 的 `skillCoverage` 从"纯自评"改为"自评 × 该技能牌组的近期复习正确率"，或新增 `evidenceLevel` 字段，逐步降低自评权重。

> 关键认知：**P0 不是从零造功能，是"接一根断掉的线"**。AI 端已经在产出牌组结构，落库 + 关联 + 跳转即可让整个差异化故事成立。

## 7. 打磨清单（P4，提升"成熟感观感"）

- **Onboarding aha**：首次登录引导「粘贴一份 JD → 30 秒看到定制学习计划」，而不是空驾驶舱。
- **空状态**：每个空列表给"下一步该做什么"的引导卡 + 行动按钮。
- **品牌**：统一图标语言、给一个真 logo、登录页副标题"面试记忆引擎"排版修正（截图里偏挤）。
- **倒计时压强**：有面试日期时，驾驶舱用"距面试 N 天 · 今日最小必做"制造节奏感。
- **统计深度**：遗忘曲线、就绪度随时间曲线、预测留存 vs 实际（FSRS 校准）。
- **暗色/密度**：复习页大量留白，可增加进度感（本组进度、连击、预计剩余时间）。

## 8. 风险与取舍

- **AI 生成卡质量**：自动造卡可能产出"识别型"而非"回忆型"卡片，拉低留存。→ 加一步"5 分钟人工过卡"或 AI 自检 atomicity。
- **成本/隐私**：模拟面试 + 批量造卡会放大 LLM 调用量；语音点评涉及录音，单人自用可本地化或显式开关。
- **别过度通用化**：坚持"单人深度好用"，先把*你自己*的 offer 闭环跑通，再谈是否对外。
- **不做的事**：不做 Final Round 那种"实时面试外挂"（伦理风险），只做赛前训练。
